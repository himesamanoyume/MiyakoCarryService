宫子护航店
宫子护航店 (MiyakoCarryService) 是一个生成AI队友的模组。

 
SPT Forge | Github

食用方式

如果你找不到左下角的邀请至队伍按钮, 请检查你是否安装了UI Fixes, 若有，则需要按F12到配置管理器中顶部打开Advanced Settings, 找到UI Fixes的Interface -> Show Group Panel, 开启即可。

  

Q&A

  • Q: 宫子护航店 的相关设定

  • A: 本文会使用 Mcs 作为 宫子护航店（Miyako Carry Service）的简称；真人玩家会被称为 老板，Mcs 生成的AI队友会被称为 护航，AI Boss队友会被称为 护航Boss，月雪宫子 会被称为 店长，本作者 会被称为 制作人（Producer、普罗丢撒），本人的 Discord频道 会被称为 Rabbit1电竞（Rabbit1 Gaming、R1G），赞助者 会被称为 打手，老板下单时就会从打手中随机抽取来作为护航生成，只要你成为过至少一次赞助者，你就可以加入到打手名单中，打手名单会在合适的时候更新

  • Q: 宫子护航店 有什么功能

  • A: 支持通过护航库存模式完全自定义护航的装备、支持生成任意类型的AI作为护航，甚至可以是其他Mod提供的AI类型、支持护航拾取指定战利品、支持通过指令系统命令护航、支持护航使用SAIN的战斗逻辑、全部功能支持Fika联机使用

服务端

  1. 官方提供中/英/俄三种本地化语言，其他语言皆显示英文。 但需要确保您已正确安装MiyakoCarryServiceServer，否则无法加载本地化。

  2. 服务端配置项（需要先安装Mcs后启动一次服务端生成配置文件）

    3. BalanceRestriction 平衡性调整。默认为false。当开启后将无法使用打开护航背包指令，且所有护航的带入战局物品都将附加消失诅咒附魔，当护航死亡时其带入的物品无论处于何处，都会立即消失
    CheckUpdate 是否检查版本更新。默认为true。
    CheckIfdian 是否检查赞助者名单更新。默认为true。
    TicketPricePerPercent 罚单每百分比价格。用于申请罚单上缴罚款来消除涨价惩罚时的定价，单位为卢布/百分比，默认为300000卢布每百分比。
    PunishmentMultiMax 涨价惩罚上限。单位为倍率，默认为1，即100%。例如值为5，则涨价惩罚上限将为500%。
    OrderPendingPaymentTime 订单待付款时间。下单护航、下发罚单时宫子商人提供的行动任务过期时间。单位为秒，默认为900秒，即15分钟。
    CompensationPrice 赔偿价格。当护航误杀老板时会进行赔偿。单位为卢布，默认为300000卢布。
    CarryServiceLevelPrice 护航等级基准价格。共5级，单位为卢布，每个等级分别设有上下限

如何自定义护航装备

  1. 你需要有已加为好友的护航
  2. 从底部栏左下角点开邀请至队伍界面
  3. 对着护航右键弹出右键菜单，选择打开库存
  4. 此时将进入护航库存模式
  5. 你可以在护航库存模式下通过与正常游玩时一模一样的操作来自定义护航的装备
  6. 进入护航库存模式时，宫子商人将提供全物品1卢布价格购买
  7. 若要返回主角色，则有两种方式：一是同样通过邀请至队伍界面对着护航右键弹出右键菜单，选择返回主角色；二是直接点击底部栏的绿色护航库存模式按钮，也一样可以返回主角色
  8. 温馨提示：护航库存模式修改的是护航在PMC模式下的装备，总所周知，PMC模式和SCAV模式是两个不同的角色，按照游戏设定SCAV角色你无法控制其装备，因此Mcs也会保留这个设定

如何续订和结算

  1. 同样从底部栏左下角点开邀请至队伍界面
  2. 同样对着护航右键弹出右键菜单，选择续订
  3. 宫子商人将会派发一个同样时长的新订单任务，将其再次完成即可延长时间
  4. 如果对着已过期的护航选择结算，那么此订单下的护航的所有数据都会被永久删除
  5. 已过期的护航的数据仍会一直保留，直到你进行结算，你可以随时对任意护航进行续订，让已过期的护航重新可用或是延长未过期护航的过期时间
  

战局中如何获取护航拾取的战利品

  1. 当护航拾取了目标战利品后，要拿护航身上的东西除了护航死亡后去翻尸体，还有就是通过Mcs指令系统中对单个护航使用"打开背包"指令，即可在护航存活状态下远程打开护航的背包。
  2. If you want it, then you'll have to take it, as you have already known.
  

如何自定义护航类型

  1. 先启动一次服务端，MiyakoCarryServiceServer/configs下会生成spawntype.json
  2. 接下来的内容如果不太懂的建议别自己乱动，修改错误会导致报错，如果实现想改，再往下看。以及你可以交给AI来做，如果还是改错了，那就把spawntype.json删了，然后重启服务端，会再次重新生成一份新的文件
  3. 首先你需要知道Json的格式规则，最好使用类似VS Code的编辑器能够对Json格式进行检查
  4. 然后你需要知道第三方Mod提供的，或者原版自带的AI的WildSpawnType名称
  5. 拿BlackDiv Mod举例，其提供了一个黑色军团的WildSpawnType：blackDivAssault
  6. 那么现在你就要在spawntype.json里以正确的格式多添加一个类型
    7. {     "WildSpawnType": "sectantOni",     "IsBoss": true,     "DisplayName": "Oni" }, // 上面是默认自带的，往后新加类型的时候要在 `}` 后带上 `,` {     "WildSpawnType": "blackDivAssault", // 这里写上WildSpawnType的名称     "IsBoss": false, // 是否作为Boss类型，使用help指令时就会为这个类型添加`[Boss]`的前缀     "DisplayName": "Black Division Assault"     // 展示名称，使用help指令时就只会展示这个名称，而不是WildSpawnType的名称 } // 最后一个 `}` 不要带 `,`
  7. 确保json格式正确，数据也填写正确后，重启服务端，此时与宫子商人聊天界面使用help指令，列出的可用类型中就会显示出所有的类型
  8. 如果填写的自定义类型在生成时出现异常，会在服务端收到报错提醒，并改为生成默认的pmc类型护航，此时你需要检查是否填写的自定义类型有错误
